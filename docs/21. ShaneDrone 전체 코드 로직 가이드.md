## 3. 부팅 시퀀스 (`main.cpp`)

```
setup()
  ├── Serial.begin(115200)          — USB 시리얼 (최대 3초 대기)
  ├── initializeSensors()           — MPU9250 I2C 초기화
  │     └── 실패 시 → 시뮬레이션 모드 (계속 진행)
  ├── initializeMotors()            — LEDC PWM 채널 설정, 최소 출력
  ├── initializeCommunication()
  │     ├── initializeSBUS()        — Serial2, 100kHz, 반전
  │     ├── initializeWiFi()        — WiFi AP 생성 (Shane_Drone)
  │     ├── initializeVTX()         — Walksnail MSP (Serial1)
  │     └── web::begin()            — AsyncWebServer + WebSocket 시작
  └── initializeControl()           — PID 게인 초기화
```

**중요**: `WiFiServer`, `AsyncWebServer`, `AsyncWebSocket`은 전역 스태틱으로 선언하면 USB 스택이 초기화되기 전에 생성자가 실행되어 크래시 루프 발생. 반드시 포인터로 선언하고 `begin()`/`initializeWiFi()` 내에서 `new`로 생성해야 함.

## 4. 센서 모듈 (`sensors.cpp`)

### 초기화 흐름
```
initializeSensors()
  ├── Wire.begin()                  — 인자 없이 호출 (기본 GPIO11=SDA, GPIO12=SCL)
  ├── 0x68 주소로 WHO_AM_I 레지스터 확인
  ├── 실패 시 0x69 주소로 재시도
  ├── 성공 시 → 레지스터 설정 (자이로 ±2000dps, 가속도 ±8g, DLPF 41Hz)
  └── 실패 시 → simulationMode = true
```

### 센서 읽기 (`readSensorData`)
```
readSensorData()
  ├── [시뮬레이션 모드] → sin/cos 기반 가상 roll/pitch/yaw 생성
  └── [정상 모드]
        ├── I2C로 가속도/자이로/자기계 raw 데이터 읽기
        ├── 오프셋 보정 (캘리브레이션 값 적용)
        └── 상보 필터로 roll/pitch 계산
              roll  = α × (roll + gyroX×dt) + (1-α) × accelRoll
              pitch = α × (pitch + gyroY×dt) + (1-α) × accelPitch
              yaw   = yaw + gyroZ×dt  (자기계 미사용)
```

### 상보 필터 파라미터
- `α = 0.98` — 자이로 98% + 가속도계 2% 융합
- `DLPF = 41Hz` — 고주파 노이즈 차단

### 알려진 문제
MPU9250 I2C가 물리 하드웨어에서 동작하지 않음 (두 모듈 모두 실패).
현재 **시뮬레이션 모드**로 동작 중.
의심 원인: NCS 핀이 3.3V에 연결되지 않거나 I2C 풀업 저항 누락.

## 5. 제어 모듈 (`control.cpp`)

### 모터 초기화
- ESP32 LEDC 주변장치 사용 (analogWrite 불가)
- 4채널, 50Hz PWM, 16비트 해상도
- 핀 매핑:

| 채널 | 핀 | 위치 |
|------|-----|------|
| 0 | GPIO5 | Front Left (FL) |
| 1 | GPIO6 | Front Right (FR) |
| 2 | GPIO7 | Rear Left (RL) |
| 3 | GPIO8 | Rear Right (RR) |

- PWM 범위: 1000μs (최소) ~ 2000μs (최대)
- LEDC 변환: `map(μs, 1000, 2000, 1638, 6553)`

### PID 제어기 구조
```c
struct PIDController {
  float kp, ki, kd;
  float setpoint;
  float lastError;
  float integral;    // anti-windup: CONSTRAIN(-maxIntegral, +maxIntegral)
  float maxIntegral;
  float output;
};
```

### PID 게인 (BETAFPV 1102 14000KV 튜닝값)

| 축 | P | I | D | maxIntegral |
|----|---|---|---|-------------|
| Roll | 1.2 | 0.8 | 0.15 | 100 |
| Pitch | 1.2 | 0.8 | 0.15 | 100 |
| Yaw | 1.5 | 0.5 | 0.05 | 50 |

### 비행 모드

#### STABILIZE 모드 (기본)
```
목표 각도 = 조종기 입력 × MAX_ANGLE (±30°)
roll/pitch → 각도 PID
yaw → 각속도(gyroZ) PID
```

#### ACRO 모드
```
목표 각속도 = 조종기 입력 × MAX_YAW_RATE × 2 (±400°/s)
모든 축 → 각속도 PID (gyroX/Y/Z 직접 제어)
```

#### ALTITUDE_HOLD 모드
현재 STABILIZE와 동일 (바로미터 센서 추가 시 구현 예정)

### 쿼드콥터 X 모터 믹싱
```
motor_FL = throttle - roll + pitch - yaw
motor_FR = throttle + roll + pitch + yaw
motor_RL = throttle - roll - pitch + yaw
motor_RR = throttle + roll - pitch - yaw
```

### 조종기 입력 정규화
- 스로틀: 1000~2000μs → 0.0~1.0
- Roll/Pitch/Yaw: 1000~2000μs → -1.0~+1.0
- 데드밴드: ±10μs (중앙값 근처 입력 무시)

## 6. 통신 모듈 (`communication.cpp`)

### SBUS 수신기
- 포트: `Serial2`, GPIO4, 100kHz 반전 신호
- 프로토콜: 25바이트 패킷 (헤더 0x0F, 엔드 0x00)
- 16채널, 11비트 해상도 (172~1811 → 1000~2000μs 변환)
- Failsafe: 100ms 이상 패킷 없으면 자동으로 안전값 설정

### 채널 매핑
| 채널 | 기능 |
|------|------|
| CH0 | 스로틀 |
| CH1 | 롤 |
| CH2 | 피치 |
| CH3 | 요 |
| CH4 (AUX1) | ARM/DISARM (>1700 = ARM) |
| CH5 (AUX2) | 비행 모드 (<1300=STABILIZE, 1300~1700=ALT_HOLD, >1700=ACRO) |
| CH6 (AUX3) | 예비 |

### WiFi AP
- SSID: `Shane_Drone`, 비밀번호: `drone123`
- IP: `192.168.4.1`
- 텔레메트리 TCP 서버: 포트 8888

### VTX (Walksnail Avatar HD V2)
- 포트: `Serial1`, TX=GPIO17, RX=GPIO16, 115200bps
- AT 명령으로 초기화 (`AT+POWER=2`, `AT+FREQ=5800`)
- 응답 없으면 조용히 실패 (시스템 계속 동작)

## 7. 웹 서버 모듈 (`web_async.cpp`)

### 아키텍처
- `ESPAsyncWebServer` (비동기, 태스크 블로킹 없음)
- `AsyncWebSocket` (실시간 양방향)
- `DNSServer` (캡티브 포털 — Shane_Drone 접속 시 자동 리디렉션)
- `LittleFS` — HTML/CSS/JS 파일 서빙

### HTTP 엔드포인트
| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/` | index.html (LittleFS) |
| GET | `/api/telemetry` | JSON 텔레메트리 1회 응답 |
| POST | `/api/command` | JSON 명령 수신 |
| WS | `/ws` | WebSocket 실시간 통신 |

### WebSocket 텔레메트리 (50Hz push)
```json
{
  "attitude": { "roll": 0.0, "pitch": 0.0, "yaw": 0.0 },
  "gyro": { "x": 0.0, "y": 0.0, "z": 0.0 },
  "motors": { "fl": 1000, "fr": 1000, "rl": 1000, "rr": 1000 },
  "armed": false,
  "battery": 0.0,
  "mode": "STABILIZE",
  "input": { "throttle": 0.0, "roll": 0.0, "pitch": 0.0, "yaw": 0.0 },
  "rcEnabled": false
}
```

### WebSocket 수신 명령
```json
{ "command": "CALIBRATE" }
{ "command": "RESET_PID" }
{ "command": "EMERGENCY_STOP" }
{ "command": "ENABLE_WEB_RC" }
{ "command": "DISABLE_WEB_RC" }
{ "rc": { "throttle": 0.5, "roll": 0.1, "pitch": 0.0, "yaw": 0.0 } }
```

### Web RC 모드
`ENABLE_WEB_RC` 명령 활성화 시 SBUS 입력 무시, WebSocket의 `rc` 값으로 드론 제어.
500ms 이상 RC 입력 없으면 자동으로 스틱 중앙 처리.

### LittleFS 마운트 전략
여러 basePath/label 조합으로 순서대로 시도:
```
basePaths = ["/spiffs", "/littlefs", "/"]
labels    = ["spiffs", FS_LABEL, null, "littlefs"]
```

## 8. ARM/DISARM 로직

```
ARM 조건 (모두 충족 시):
  ✓ AUX1 > 1700 (ARM 스위치 ON)
  ✓ 스로틀 < 1100 (최소 스로틀)
  ✓ Roll/Pitch/Yaw 중앙 ±50μs 이내
  ✓ 배터리 > 3.3V
  ✓ systemReady == true

DISARM 조건:
  - AUX1 < 1700 (ARM 스위치 OFF)
  - 배터리 < 3.4V (긴급 자동 DISARM)
  - SBUS 수신 타임아웃 1초 초과 → Failsafe 모드
```

## 9. 배터리 모니터링

- 측정 핀: `A0`, 분압비 3.0 적용
- EMA 필터 (α=0.15)로 노이즈 제거
- 0.2V 미만 측정값은 노이즈로 간주 → 이전 값 유지

| 전압 | 동작 |
|------|------|
| > 3.5V | 정상 |
| 3.4~3.5V | 경고 메시지 (10초 간격) |
| < 3.4V | 즉시 자동 DISARM |

| 시나리오 | 처리 |
|----------|------|
| SBUS 수신 타임아웃 (1초) | Failsafe: 스로틀 1200, 스틱 중앙 → 10초 후 DISARM |

## 12. 업로드 방법

### 펌웨어 (JTAG — 권장)
```bash
pio run -e arduino_nano_esp32-jtag -t upload
```
- COM4 불필요, USB JTAG 인터페이스 사용
- 사전 조건: Zadig로 `USB JTAG/serial debug unit (Interface 0)`에 WinUSB 드라이버 설치
- 업로드 후 RESET 누르지 말고 **USB 재연결**로 부팅

### LittleFS (웹 파일)
```bash
pio run -e arduino_nano_esp32-jtag -t uploadfs
```
- `data/` 폴더 내용이 변경될 때마다 실행

### 시리얼 모니터
```bash
pio device monitor -p COM4 -b 115200
```

## 13. 알려진 이슈 및 해결 방법

| 이슈 | 원인 | 해결 |
|------|------|------|
| 크래시 루프 `rst:0xc` | WiFiServer/AsyncWebServer 전역 스태틱 생성자 | 포인터로 변환, `begin()`에서 `new` 초기화 |
| COM4 업로드 실패 | VS Code PlatformIO가 COM4 점유 | JTAG 업로드 방식으로 전환 |
| DFU 모드 진입 불가 | Windows WinUSB 드라이버 미설치 | Zadig로 Arduino DFU에 WinUSB 설치 |
| MPU9250 I2C 불통 | NCS 핀 플로팅 또는 풀업 저항 누락 | NCS를 3.3V에 연결, 4.7kΩ 풀업 저항 추가 |
| JTAG 업로드 후 DFU 진입 | OpenOCD 리셋 + RESET 1회 = 더블탭 인식 | USB 재연결로 부팅 (RESET 버튼 사용 금지) |
